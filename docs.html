01 · Introduction

What is MetaPulse?

MetaPulse is a browser-based meta tag auditor and social preview simulator. Paste your HTML, and in under a second you'll know exactly which tags are missing, which are misconfigured, and what your link will look like when it lands in a Slack thread, a WhatsApp chat, or a tweet.

The premise is simple: meta tags are invisible to most developers until something goes wrong. You ship a blog post, your colleague shares it on Slack, and it comes up as a sad grey box with no image. Or the title is truncated at the worst possible character. Or Twitter shows a tiny thumbnail instead of a dramatic banner.

These are not mysterious failures. Every platform has documented, predictable requirements. MetaPulse codifies those requirements into an audit engine that checks 15+ tag types against platform-specific rules — not just "does the tag exist?" but "is the value appropriate for each context?"

Who it's for

Web developers who want to verify meta coverage during development, before a PR merges. SEO specialists who need a quick visual check without hunting through raw source. Content and social media teams who want to preview what a shared link will actually look like — not guess.

Note MetaPulse is a client-side tool. Everything runs in your browser. No data is sent to any server. No accounts, no tracking, no nonsense.
02 · Getting Started

Quick Start

You're three steps away from knowing exactly why your links look bad on social.

Step 1 — Get your HTML

Open the page you want to audit in your browser. Press Cmd+U (Mac) or Ctrl+U (Windows) to open View Source. Select everything (Cmd+A) and copy it.

Alternatively, if you have access to your codebase, you can paste just the <head> section — MetaPulse only needs what's in there.

Step 2 — Paste and audit

Head to the Auditor. Make sure the HTML Snippet tab is selected. Paste your HTML into the text area, then click Audit Now (or press Cmd+Enter / Ctrl+Enter).

Step 3 — Read the results

The sidebar shows your SEO Health Score (0–100), a tag-by-tag breakdown, and an asset checklist. The main panel shows your previews. Switch between Twitter, Slack, WhatsApp, and Google using the tabs at the top.

Any tag flagged as warn or error has a Fix ↗ button. Click it to see a copy-pasteable HTML snippet with an explanation of what it does and why it matters.

Using the URL option

You can also switch to the URL tab and enter a page URL directly. MetaPulse will attempt to fetch the page via a CORS proxy. This works for most public pages, but if the target server blocks it, MetaPulse will say so clearly and switch you to HTML mode.

Tip For local development, the HTML Snippet method is always more reliable. There's no CORS to worry about, and you see exactly what's in your current build, not a cached version.
03 · Privacy

Your data stays yours.

MetaPulse is entirely client-side. The HTML you paste never leaves your browser. The audit runs locally. The previews render locally.

The only network requests MetaPulse makes are:

  • URL fetching (optional): If you use the URL tab, the page is fetched via a public CORS proxy (corsproxy.io). This is an optional convenience and can be avoided entirely by pasting HTML.
  • OG image loading: When rendering previews, your browser fetches the og:image URL directly. This is the same request a browser would make when visiting your page normally.
  • Google Fonts: Typography is loaded from Google Fonts on page load.

No analytics. No cookies. No fingerprinting. MetaPulse has no backend.

04 · Audit Engine

Basic Meta Tags

Before Open Graph existed, before Twitter Cards were a thing, there were five basic meta tags that every page should have. Surprisingly many pages still get these wrong.

<title>

The page title is the most fundamental signal in the whole meta tag universe. It appears in browser tabs, bookmark lists, Google's search result headlines, and as a fallback title on every social platform that hasn't found an og:title first.

Optimal length: 50–60 characters. Google displays up to roughly 600px of width, which typically fits 50–60 characters. Go beyond that and you'll see an ellipsis in search results. Shorter is fine. Much longer is a gamble.

Common mistake Using the same title on every page — usually the site name alone ("Acme Corp" on every single URL). Search engines see this as thin content. So does the human trying to find the right tab.

Meta description

The meta description is Google's official source for the search result snippet — though Google reserves the right to ignore it and pull text from the page body instead if it thinks that's more relevant to the query. Other platforms use it more faithfully.

Optimal length: 120–160 characters. Long enough to be informative. Short enough to not get cut off. Write it like ad copy, not a legal disclaimer.

Canonical URL

The link rel="canonical" element tells search engines which URL is the "real" version of a page. Critical when the same content is accessible at multiple URLs — with or without www, with or without trailing slashes, with or without query parameters. Without it, Google may split your ranking signals across all the variants.

Viewport meta

The viewport meta tag tells mobile browsers how to scale the page. Without it, mobile browsers typically render the page at a desktop width and then scale it down — making text tiny and interactions fiddly. Google uses mobile-friendliness as a ranking signal.

The standard value: width=device-width, initial-scale=1. Rarely needs to be anything else.

Open Graph

The Open Graph protocol was invented by Facebook in 2010 and has since become the universal language of link previews. If a platform wants to show a rich link card, it almost certainly starts by reading your OG tags.

The four required tags

The Open Graph spec nominates four "basic" tags that should be present on every page: og:title, og:type, og:image, and og:url. MetaPulse flags og:title, og:description, and og:image as required because, in practice, a link preview without any of these three is broken.

og:title

The link's display title on social platforms. Can (and often should) differ from the page <title>. Optimal: under 95 characters. Twitter truncates at 70 in its own title field.

og:description

The teaser text below the title. Optimal: under 200 characters. Write for humans who are deciding whether to click, not for keyword density.

og:image

The preview image. This single tag has the biggest impact on click-through rates of any meta tag. Requirements:

  • Must be an absolute URL (starting with https://). Relative URLs will not work across platforms.
  • Recommended dimensions: 1200×630px for Open Graph. Twitter Large Image Cards prefer 1200×600px (2:1 ratio).
  • Format: JPG, PNG, or WebP. GIF is supported by some platforms but not all. WebP is the modern choice.
  • File size: Under 300KB is a safe target. Messaging apps (WhatsApp, iMessage) sometimes refuse to load larger images.

og:url

The canonical URL for this piece of content. Platforms use this to aggregate share counts. Without it, each URL variant (with/without www, with/without trailing slash) gets its own share count.

Tip Set og:image:width and og:image:height as well. Some platforms won't wait to load and parse the image — they'll just trust what you declare.

Twitter Cards

Twitter (now X) has its own set of meta tags that override OG tags within the platform. Most of them fall back to their OG equivalents if absent — but twitter:card doesn't. Without it, you get no card at all.

twitter:card — the non-negotiable one

The twitter:card tag declares which card type to render. Valid values: summary, summary_large_image, app, player. The most useful for content pages is summary_large_image, which renders a full-width banner image above the title and description.

Without this tag, Twitter/X falls back to a plain text link. No image. No card. Just a URL.

The rest of the Twitter tags

twitter:title, twitter:description, and twitter:image all fall back to their OG equivalents when absent. Setting them explicitly lets you tailor the Twitter experience — for instance, a punchier headline that works better in a 280-character context.

twitter:site takes your @handle. It shows up in the card as attribution. It's free branding. You should set it.

07 · Audit Engine

The Scoring System

The Health Score is a weighted 0–100 number. It's opinionated. It's meant to be directional, not definitive — a useful signal, not an absolute truth.

How it's calculated

Each tag is assigned a weight based on how much it matters in practice:

  • Required tags (title, description, og:title, og:description, og:image, twitter:card) — worth 12 points each
  • Optional tags (canonical, og:url, og:type, og:site_name, twitter:*) — worth 4 points each

For each tag:

  • OK → full weight
  • Warn (present but length-exceeded or misconfigured) → 50% weight
  • Missing (optional tag absent) → 15% weight
  • Error (required tag absent) → 0 weight

The total earned weight is divided by the total possible weight and scaled to 100.

Grade thresholds

Score Grade What it means
90–100 EXCELLENT Nearly perfect meta coverage. Minor optional tags may be absent.
75–89 GOOD Solid. Required tags present. A few optional ones missing.
55–74 FAIR Some important tags absent or configured poorly.
30–54 POOR Multiple critical tags missing. Social previews will look broken.
0–29 CRITICAL Bare minimum. Links on social media will be embarrassing.
08 · Platform Previews

Twitter / X

MetaPulse renders both Twitter card variants:

Summary Card

A horizontal layout with a small square thumbnail on the left and title/description on the right. Used when twitter:card is summary. Title is capped at 70 characters; description at 125.

Large Image Card (summary_large_image)

A vertical layout with a full-width image above the metadata. The aspect ratio is roughly 2:1 (1200×600px optimal). Title capped at 70 characters. This is the card type you want for content pages — it stops thumbs mid-scroll.

MetaPulse loads the actual og:image (or twitter:image) URL from your HTML and displays it in the card preview. What you see is what your audience sees.

09 · Platform Previews

Slack

Slack's link unfurl is a horizontal card with a coloured left border, site name, title, description, and a thumbnail image. MetaPulse simulates this precisely, including Slack's signature dark background and the green-accent left border.

Slack reads og:site_name, og:title, og:description, and og:image. The thumbnail is a 80×80px crop of the OG image. If no OG image is present, the thumbnail slot is empty.

Character limits are more generous in Slack than on Twitter, but the description is still cropped after roughly 180 characters in most clients.

10 · Platform Previews

WhatsApp

WhatsApp renders a dark-themed link preview inside the message bubble. Title and description are pulled from OG tags. The preview image is displayed at 16:9 ratio above the text metadata.

WhatsApp is stricter than most platforms: the og:image must be an absolute HTTPS URL, and the image file size should be under 300KB. Larger images may silently fail to load. Relative URLs are always ignored.

Title is capped at approximately 60 characters, description at 100. WhatsApp doesn't read twitter:* tags — it's exclusively OG.

11 · Platform Previews

Google Search

The Google Search preview shows your page as it might appear in a search result — breadcrumb, title, and snippet. MetaPulse offers both desktop and mobile variants.

Google uses the <title> tag for the result headline, capped at approximately 60 characters (Google calculates by pixel width, not character count, but 60 chars is a reliable proxy). The description meta tag is used for the snippet, but Google may substitute its own excerpt from the page body if it thinks that's more relevant to a given query.

Important Google's SERP snippet is an approximation. The actual display depends on the search query, device, and Google's real-time decisions. MetaPulse shows you what you've configured, not a guarantee of what Google will show.

Image Guidelines

The og:image is the single highest-leverage meta tag you can add. Here's the spec that keeps every platform happy:

Slack 80 × 80px thumb 1:1 crop — JPG, PNG, WebP LinkedIn 1200 × 627px 1.91:1 5MB JPG, PNG
Recommendation Create one 1200×630px image. It works acceptably everywhere. If Twitter performance matters to you, additionally create a 1200×600px variant and set it as twitter:image.
14 · Reference

Code Generator

The Generated Code tab in the auditor produces a complete, formatted <head> snippet based on the values found in your HTML — or sensible placeholders where tags are missing. Copy it directly into your codebase.

The generated code includes:

  • All primary meta tags (charset, viewport, title, description, robots, canonical)
  • Full Open Graph block (type, title, description, image with dimensions, url, site_name)
  • Full Twitter Cards block (card, title, description, image, site handle)
  • Inline comments grouping each section and explaining where to find missing values

The Fix modal (opened via the Fix ↗ button on any flagged tag) provides a smaller, tag-specific snippet with a contextual explanation. These are designed to be minimal, targeted patches — paste them into an existing <head> rather than replacing it wholesale.
